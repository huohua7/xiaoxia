# 🦐 小虾 XiaoXia

一套多Agent的研究生文书系统，喂给Claude就能跑。

## 背景

去年我们开始琢磨一件事：做了六年留学文书，带了四百多个学生，我们几个人脑子里那些做判断的直觉，到底能不能教给AI。

先说这个直觉是什么。

我们平时给学生做文书，最花时间的环节不是写，是前面的访谈。一个学生过来，简历上写了三段经历，你让他自己讲他就开始背简历。但你要是换个问法，问他"你在实验室的时候一般几点到，到了先干嘛"，他就开始说真话了。说着说着你会发现他提到某件事的时候突然特别具体，开始说人名、说时间、说"我当时特别气"。这种时候基本上挖下去就有东西。

去年有个学化学的学生要转CS，简历上素材很薄。聊完发现他之前在实验室自己写过python脚本批量处理光谱数据，嫌手动太慢自己学着搞的。简历上没写因为他觉得那脚本太糙了。但这个素材比他简历上任何一条都好用。

类似的判断我们每天都在做，但我们自己也说不太清楚到底在凭什么判断。去年下半年开始我们四个人互相逼，"你说这段经历你觉得能用，凭什么？你脑子里到底在算什么？"逼了三个月确实逼出来一些东西，比如判断一段素材值不值得挖，最靠谱的指标其实不是这段经历多牛，是学生讲到这的时候有没有突然变具体。

我们把这些东西整理了一下，写成了一套Claude能跑的配置文件。七个Agent各管一摊，素材挖掘的、做策略的、写文书的、审核的、查一致性的、准备面试的，加一个查项目信息的。不是那种"帮我写篇PS"的prompt，是一整条流水线。

放出来的原因也简单。这行说白了赚的是信息差的钱。同一个学校同一个专业的两个学生，一个花了三万找我们，PS质量就是比另一个自己写的好一截，但这个差距跟两个人本身的水平没什么关系。方法论层面的东西不应该有门槛。我们接单照接，真人顾问能给的东西AI短期内给不了（我知道哪个ad今年口味变了、哪个教授的邮箱发过去大概率石沉大海），但至少别让一个学生因为没钱，连PS开头不能用名言引用都不知道。

## 系统

整套流程大概这么走的：

上来先做个诊断，你的背景对着目标项目是什么位置，哪个是冲刺哪个是保底。同时另一个Agent去扒目标项目的招生信息、教授方向这些。

然后是素材挖掘，这步最重要。一个Interviewer Agent跟学生聊，我们设计了三个阶段：先把所有经历快速过一遍看哪段温度高，然后重点经历逐个挖，最后在所有经历之间找暗线。我们管这叫"连点成线"，好的PS都有这么一条贯穿全文的东西。

这个Interviewer做了不少细节。提问用的是我们说的"脚手架式"，不问大问题，问小的、具体的，学生答起来没压力。还有个东西叫"细节密度异常检测"，说白了就是学生说自己做了什么大事但追问两轮啥细节都给不出来，系统会标一下，后面写文书的时候自动把这段素材的权重降下来。

素材挖完了一个Strategist Agent做叙事策略，决定核心主线是什么、素材怎么拆成模块、每篇文书用哪几个模块、从什么角度切。申多个学校的话还会做跨项目分配，保证不同学校的文书之间差异够大。这步还有个点，系统会看推荐信已经覆盖了什么维度，PS就避开那些维度，两边不重复。

然后才写。支持PS、SOP、Diversity、Research Statement、短问答。风格会参考学生自己的语言习惯做校准。

写完审核，一个Agent从招生官视角审读，另一个查多篇文书之间有没有事实打架。学生自己也同时看。三方意见合起来决定改不改。

审核这块说一下AI痕迹检测。我们一开始就做了个禁用词清单，"ignited my passion"那些。后来发现没用，光禁词解决不了问题。你把那些词全换了，文书读起来还是像AI写的。后来想明白了，问题在结构：每段差不多长，每个论点恰好配一个例子，情感从头到尾平滑上升。人写东西不会这么整齐。所以现在做两层检测，结构层权重比词汇层高。

最后一步是面试准备。一个Agent预判招生官读完你文书之后会追问什么，哪些点你自己讲得清楚哪些点你可能露馅。

## 文件

```
├── CLAUDE.md                    # 核心配置，所有规则都在这
├── CLAUDE_CODE_PROMPT.md        # Claude Code的操作手册
├── banned_expressions.md        # 禁用表达
├── walkthrough_test.md          # 端到端测试记录
├── agents/
│   ├── interviewer.md           # 素材挖掘，最长的一个
│   ├── program_researcher.md    # 项目调研
│   ├── strategist.md            # 策略
│   ├── writer.md                # 写
│   ├── reviewer.md              # 审核
│   ├── consistency_checker.md   # 一致性
│   └── interview_coach.md       # 面试准备
└── projects/                    # 用户数据，不传
```

## 用法

**Claude Code**：克隆仓库，`CLAUDE_CODE_PROMPT.md` 内容丢进CLAUDE.md，直接聊。

**Claude Project**：`CLAUDE.md` 加 `agents/` 下面所有文件加 `banned_expressions.md` 传到Project Knowledge。Instructions 写一句让它按配置文件工作就行。

**System Prompt**：塞不下就挑着放。`CLAUDE.md` 先放，然后 `interviewer.md`，然后 `writer.md` 加 `banned_expressions.md`。

## 局限

模型写英文的能力就是上限，英文好的自己写可能更好。行业内部信息AI拿不到。学生不配合的话系统也没招。

## 迭代

第一版跑了几个case发现一堆问题，改了。然后自己做对抗测试故意搞破坏，又改了。最后拿虚构申请人端到端走了一遍。过程记在 `walkthrough_test.md` 里。
